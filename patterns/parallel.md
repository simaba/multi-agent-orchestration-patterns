# Parallel Pattern

A parallel pattern runs multiple agents at the same time and combines their outputs.

## Structure
Several agents work independently and an aggregator combines or compares the results.

## Best use cases
- diversified analysis
- independent retrieval tasks
- multiple candidate generation

## Strengths
- faster throughput in some workflows
- broader coverage
- useful for cross-checking

## Risks
- aggregation complexity
- duplicated effort
- conflicting outputs if evaluation is weak
